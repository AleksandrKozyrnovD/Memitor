# Модуль ядра для мониторинга памяти процессов

Данный модуль предоставляет три интерфейса в `/proc` для анализа памяти процессов:
- чтение произвольной области памяти;
- детальный просмотр карты памяти (VMA) с информацией о резидентных страницах;
- отслеживание системных вызовов `brk` и `mmap` с записью изменения RSS.

Модуль загружается как единый, все файлы появляются в `/proc` после загрузки.

---

## /proc/memitorreader – чтение памяти процесса

**Назначение**  
Позволяет прочитать указанное количество байт из адресного пространства заданного процесса.

**Как использовать**  
1. Записать в файл строку вида:  `PID ADDR BYTES`
- `PID` – идентификатор процесса (десятичное число);
- `ADDR` – начальный адрес (шестнадцатеричный, с префиксом `0x`);
- `BYTES` – количество байт для чтения (десятичное, не более 4096).

Пример:  
```bash
echo "1234 0x7f123456 64" > /proc/memitorreader
```
2. Прочитать файл
```bash
cat /proc/memitorreader
```

Таким образом, выдается
содержимое памяти по указанному адресу (максимум 4096 байт). Если чтение невозможно, возвращается ошибка.

## /proc/memshot – карта памяти процесса с резидентными страницами

**Назначение**  
Выводит информацию о регионах виртуальной памяти (VMA) для заданного PID, аналогично /proc/<pid>/maps, но дополняет каждый регион с флагами rw списком непрерывных диапазонов физически присутствующих (резидентных) страниц.

**Как использовать**  
1. Записать PID
```bash
echo 1234 > /proc/memshot
```
2. Прочитать файл
```bash
cat /proc/memshot
```

Результат выдается в таком виде:
```
start-end [pages] rwxp/s pgoff major:minor inode path
```
где
- start-end – диапазон адресов;
- [pages] – количество страниц в регионе;
- rwxp/s – флаги доступа и тип разделения;
- pgoff – смещение в файле (если есть);
- major:minor – устройство;
- inode – номер инода;
- path – путь к файлу или [heap], [stack], [anon].

Сразу после этой строки (для регионов с rw) выводится дополнительная информация о резиденции:
- если есть резидентные страницы – строки вида:

```
	start-end [pages]
```
  где каждая строка описывает непрерывный диапазон присутствующих страниц внутри VMA;
- если страниц нет – выводится:
```
	[no resident pages]
```

## /proc/memitor – мониторинг вызовов brk/mmap и RSS

**Назначение**
Отслеживает системные вызовы brk и mmap для указанного процесса, записывая момент вызова, счётчики вызовов и текущее количество резидентных страниц (RSS). Это позволяет наблюдать динамику изменения кучи и отображаемых областей.

**Как использовать**  
1. Записать PID
```bash
echo 1234 > /proc/memitor
```
После этого модуль начинает перехватывать вызовы brk и mmap для данного процесса.
2. Прочитать файл
```bash
cat /proc/memitor
```

Результат выдается в виде таблицы с колонками
```
timestamp seq brk_count mmap_count resident_pages
```
где
- timestamp – время события в формате сек.мкс (например, 123456.789012);
- seq – порядковый номер события (глобальный счётчик);
- brk_count – общее количество вызовов brk с момента начала мониторинга;
- mmap_count – общее количество вызовов mmap с момента начала мониторинга;
- resident_pages – количество резидентных страниц (RSS) процесса на момент события.
